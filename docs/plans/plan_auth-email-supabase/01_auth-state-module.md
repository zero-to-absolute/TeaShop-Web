# Фаза 1: Модуль auth-state.js

**Зависимости:** нет (первая фаза)
**Основание:** ADR-3 (централизация auth-состояния)

---

## Что реализуется

Центральный модуль управления состоянием авторизации. Все остальные модули
(login, register, profile, header) будут импортировать из него — ни один из них
не вызывает `supabase.auth` напрямую кроме своих специфических операций.

Модуль создаёт **единственную подписку** на `onAuthStateChange` для всего приложения.

## Функции и экспорты модуля

### `getCurrentUser()`
- **Возвращает:** `Promise<User | null>`
- **Описание:** Получает текущего пользователя через `supabase.auth.getSession()`.
  Возвращает `session.user` или `null` если сессии нет.
- **Обработка ошибок:** `try/catch`, при ошибке возвращает `null` и логирует в консоль.

### `onAuthChange(callback)`
- **Принимает:** `callback(user)` — функция, принимающая `User | null`
- **Возвращает:** объект `{ unsubscribe }` для отписки
- **Описание:** Оборачивает `supabase.auth.onAuthStateChange`. Вызывает `callback`
  при каждом изменении (вход, выход, обновление токена). Передаёт `user` или `null`.

### `logout()`
- **Возвращает:** `Promise<void>`
- **Описание:** Вызывает `supabase.auth.signOut()`. При ошибке логирует в консоль.
  После выхода `onAuthStateChange` срабатывает автоматически — подписчики обновятся сами.

## Критерии приёмки

- [ ] Файл `src/js/auth/auth-state.js` создан
- [ ] Модуль использует ES module синтаксис (`import`/`export`)
- [ ] Все три функции экспортированы именованно
- [ ] Импортирует `supabase` из `../shared/supabase.js` (не из config.js напрямую)
- [ ] Комментарии на русском, имена на английском
- [ ] Каждый Supabase-вызов обёрнут в `try/catch`

## Как проверить руками

Открыть браузерную консоль на любой HTML-странице с подключённым Supabase CDN
и импортом модуля. Либо создать временный `test.html` со скриптом:
```html
<script type="module">
  import { getCurrentUser } from './src/js/auth/auth-state.js';
  const user = await getCurrentUser();
  console.log('Текущий пользователь:', user); // null, если не залогинен
</script>
```
Ожидаемый результат: `null` (ни один пользователь ещё не создан).
