# Фаза 2: Страница входа

**Зависимости:** Фаза 1 (auth-state.js)
**Основание:** ADR-4 (redirect через query-параметр)

---

## Что реализуется

Рабочая страница входа по email и паролю. После этой фазы можно:
- открыть `login.html` в браузере
- ввести email + пароль
- войти в аккаунт (или получить понятное сообщение об ошибке)
- быть перенаправленным после успеха

Также создаётся базовый `auth.css` с общими стилями форм авторизации.

## Логика `login.js`

### Инициализация
- Импортирует `supabase` из `../shared/supabase.js`
- Импортирует `getCurrentUser` из `./auth-state.js`
- При загрузке страницы проверяет: если пользователь **уже залогинен** (`getCurrentUser()` → не null) — сразу редирект (не показывать форму входа авторизованному)

### Обработчик формы `login-form` (событие `submit`)
1. `event.preventDefault()`
2. Читает `email`, `password` из полей
3. Клиентская валидация:
   - email не пустой
   - password не пустой
4. Блокирует кнопку submit (предотвращает двойной клик)
5. Вызывает `supabase.auth.signInWithPassword({ email, password })`
6. При ошибке: отображает `error.message` в `#error-message`, разблокирует кнопку
7. При успехе: читает `?redirect=` из `window.location.search`, редиректит туда или на `/`

### Функция `showError(message)`
- Показывает элемент `#error-message`
- Устанавливает текст

### Функция `hideError()`
- Скрывает элемент `#error-message`

## CSS `auth.css` — что включает эта фаза

Базовые стили, общие для всех auth-страниц:
- Центрирование формы на странице (flexbox)
- Стиль карточки формы (белый фон, тень, скругление)
- Стили `<input>` (border, padding, focus-ring)
- Стиль кнопки submit (цвет бренда, hover, disabled-состояние)
- Стиль `#error-message` (красный текст, скрыт по умолчанию через `display: none`)
- Медиа-запрос: от 375px (форма на всю ширину) до 768px+ (карточка 400px)

## Критерии приёмки

- [ ] `login.html` открывается в браузере без ошибок в консоли
- [ ] Форма отображается по центру страницы на ширине 375px и 1280px
- [ ] При попытке войти с пустыми полями — показывается ошибка (клиентская)
- [ ] При вводе неверных данных — отображается сообщение об ошибке от Supabase
- [ ] При верных данных — происходит редирект (на `/` если нет `?redirect=`)
- [ ] Если пользователь уже залогинен и открывает `/login.html` — редирект на `/`
- [ ] Кнопка блокируется во время запроса (нет двойного сабмита)
- [ ] Ссылка «Зарегистрируйтесь» ведёт на `register.html`

## Как проверить руками

1. Открыть `src/pages/login.html` в браузере
2. Попробовать отправить пустую форму → увидеть ошибку
3. Ввести несуществующий email → увидеть ошибку от Supabase
4. Зарегистрировать тестового пользователя вручную через Supabase Dashboard
5. Войти с верными данными → редирект на `/`
6. Открыть `login.html` снова (уже залогинен) → автоматический редирект
