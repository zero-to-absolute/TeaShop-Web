# Фаза 4: Модуль навигации header.js

**Зависимости:** Фазы 1–3 (auth-state.js, login.html, register.html)
**Основание:** ADR-2 (JS-модуль для auth-зависимого UI)

---

## Что реализуется

Универсальный модуль `shared/header.js`, который подключается на **каждой** странице
и автоматически переключает навигацию в зависимости от auth-состояния:
- **Гость:** видит ссылки «Вход» и «Регистрация»
- **Авторизованный:** видит ссылки «Профиль» и «Выход»

После этой фазы страницы `login.html` и `register.html` получают рабочую навигацию.
Кнопка «Выход» реально выходит из системы.

## HTML-разметка `<header>` (добавляется в login.html и register.html)

Структура header содержит **оба** варианта навигации сразу. JS только переключает видимость:

```
<header id="site-header">
  <a href="/" class="logo">TeaShop</a>

  <!-- Навигация для гостей -->
  <nav id="guest-nav">
    <a href="login.html">Вход</a>
    <a href="register.html">Регистрация</a>
  </nav>

  <!-- Навигация для авторизованных -->
  <nav id="user-nav" style="display: none;">
    <a href="profile.html">Профиль</a>
    <button id="logout-btn">Выход</button>
  </nav>
</header>
```

Начальное состояние: `guest-nav` виден, `user-nav` скрыт (`display: none`).
`header.js` обновит это при загрузке и при изменении auth-состояния.

---

## Логика `header.js`

### Инициализация (вызывается автоматически при импорте)
- Импортирует `getCurrentUser`, `onAuthChange`, `logout` из `../auth/auth-state.js`

### Функция `updateHeader(user)`
- Если `user !== null`: скрыть `#guest-nav`, показать `#user-nav`
- Если `user === null`: показать `#guest-nav`, скрыть `#user-nav`

### При загрузке страницы (`DOMContentLoaded`)
1. Получает текущего пользователя: `getCurrentUser()`
2. Вызывает `updateHeader(user)` — немедленно обновляет видимость
3. Вешает обработчик на `#logout-btn`:
   - Вызывает `logout()`
   - После выхода `onAuthStateChange` сработает → `updateHeader(null)` → UI обновится
   - Редирект на `/`
4. Подписывается на `onAuthChange(updateHeader)` — для реактивного обновления при входе/выходе на других вкладках

### Экспорт
Модуль ничего не экспортирует — он работает как side-effect модуль.
Достаточно `import '../shared/header.js'` в скрипте страницы.

## Изменения в login.js и register.js

В каждый файл добавить в начало:
```js
import '../shared/header.js';
```

Альтернатива: подключить `header.js` отдельным тегом `<script type="module">` в HTML.
**Выбрать вариант через import** — это правильнее для ES modules архитектуры.

## Изменения в auth.css

Добавить стили для `<header>`:
- Flex-строка с logo слева, nav справа
- Стиль ссылок nav
- Стиль кнопки `#logout-btn` (похожа на ссылку, не на button)
- Адаптив: от 375px (nav под logo) до 768px+ (nav в строку)

## Критерии приёмки

- [ ] `header.js` создан в `src/js/shared/`
- [ ] На `login.html`: видна навигация с «Вход» и «Регистрация» (не залогинен)
- [ ] На `register.html`: видна та же навигация гостя
- [ ] После входа через `login.html` (редирект): если вернуться на страницу с header — видна навигация авторизованного
- [ ] Кнопка «Выход» успешно выходит из системы
- [ ] После выхода навигация переключается на гостевую без перезагрузки страницы
- [ ] Логотип «TeaShop» ссылается на `/`

## Как проверить руками

1. Открыть `login.html` — проверить видимость гостевой навигации
2. Войти → редирект → вернуться на `login.html` → убедиться в редиректе (уже залогинен)
3. Открыть `register.html` — проверить гостевую навигацию
4. Открыть DevTools → Application → Local Storage → убедиться, что сессия есть
5. Нажать «Выход» (если есть страница с user-nav) → проверить очистку сессии
6. Открыть две вкладки: войти на одной → на другой header должен обновиться при следующем взаимодействии
